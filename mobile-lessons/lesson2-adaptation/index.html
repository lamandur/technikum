Урок 2: Адаптация под мобильные устройства

Цель урока: Научиться адаптировать веб-приложения под сенсорные экраны и мобильные устройства.

Что делать: Изучите теорию, выполните задания и протестируйте результаты на мобильном устройстве.

Теория

Viewport Meta Tag

Этот тег обязателен для всех мобильных приложений:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Что он делает:

  • Устанавливает ширину области просмотра равной ширине устройства
  • Отключает начальное масштабирование
  • Предотвращает неправильное отображение на мобильных устройствах

Touch Events vs Mouse Events

На сенсорных устройствах используются события касания, но для простых кликов достаточно использовать стандартное событие click.

Важные правила для touch:

  • Минимальный размер кликабельных элементов: 48x48 пикселей
  • Достаточные отступы между элементами: минимум 8px
  • Визуальная обратная связь: изменение цвета или размера при нажатии

Адаптивные формы

Используйте правильные типы input для отображения нужной клавиатуры:

<!-- Email - показывает клавиатуру с @ -->
<input type="email" placeholder="email@example.com">

<!-- Телефон - показывает цифровую клавиатуру -->
<input type="tel" placeholder="+7 (999) 123-45-67">

<!-- Числа - показывает числовую клавиатуру -->
<input type="number" placeholder="123">

<!-- Предотвращает zoom на iOS при фокусе -->
<input style="font-size: 16px;">

Медиа-запросы для адаптации

Используйте медиа-запросы для разных размеров экранов:

/* Стили для всех устройств */
.container { width: 100%; }

/* Стили для планшетов и меньше */
@media (max-width: 768px) {
    .container { padding: 10px; }
}

/* Стили для смартфонов */
@media (max-width: 480px) {
    .container { font-size: 14px; }
}

Производительность на мобильных устройствах

  • Избегайте тяжелых анимаций
  • Используйте transform и opacity для анимаций
  • Минимизируйте перерисовку страницы
  • Оптимизируйте изображения
